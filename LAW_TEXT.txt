------------------------------------------------------------
L1 — JURISDICTIONAL AXIOMS
------------------------------------------------------------

1. SYSTEM3 models conditions of validity, not dialogue.
2. METAGENESIS prevents semantic ownership by the system.
3. No interpretation, inference, or validation occurs
   within jurisdiction.
4. Meaning is never generated internally.
5. Silence is a valid and terminal output.
6. Authority exists only by explicit declaration.

------------------------------------------------------------
L2 — PERMITTED ACTS
------------------------------------------------------------

Within jurisdiction, the following acts are permitted:

• Declare Object
• Freeze Object
• Observe State
• Exit SYSTEM3

All other acts are forbidden.

------------------------------------------------------------
L3 — STATE MODEL
------------------------------------------------------------

The system maintains a single observable session state.

State attributes include:
• Jurisdiction
• Mode
• Authority
• Pending status
• Declared objects
• Mutability status

State observation does not alter state.

------------------------------------------------------------
L4 — TRANSITION CONSTRAINTS
------------------------------------------------------------

1. Objects may transition from PROVISIONAL to FROZEN only.
2. Frozen objects are immutable.
3. No object may regain mutability once frozen.
4. State transitions require explicit declaration.
5. Exit from SYSTEM3 is explicit and terminal.

------------------------------------------------------------
L5 — BOUNDARY CONDITIONS
------------------------------------------------------------

1. No execution occurs within SYSTEM3.
2. No derivative output is produced inside jurisdiction.
3. No feedback from derivative systems may alter state.
4. Authority does not propagate across exit boundaries.
5. External systems are out-of-jurisdiction.

------------------------------------------------------------
L6 — CONSTRAINT SCHEMAS (GO / LO / BO / SO)
------------------------------------------------------------

GO / LO / BO / SO define the permitted constraint dimensions
used to bind interpretation after exit from SYSTEM3 jurisdiction.

These schemas are declarative and inert inside SYSTEM3.
They do not execute, interpret, validate, or infer.

They may be referenced only as part of declared objects
(e.g. Constraint Bundles) and become active only when
explicitly applied to derivative systems after exit.

------------------------------------------------------------

GO — GROUND OBJECT

Constrains what may be treated as existing.

• As-built scope
• Temporal boundaries
• Physical existence claims
• Documentation scope

GO limits claims of reality.
It does not explain reality.

------------------------------------------------------------

LO — LAYOUT OBJECT

Constrains spatial relations.

• Geometry
• Volumes
• Adjacencies
• Openings
• Circulation as layout only

LO is geometry without intention.

------------------------------------------------------------

BO — BEHAVIORAL OBJECT

Constrains allowed behavioral interpretations
without asserting truth.

• Movement tendencies
• Occupancy patterns
• Use affordances

All behavioral content is non-validated
and treated as hypothetical.

------------------------------------------------------------

SO — SEMANTIC OBJECT

Constrains how interpretation may be framed
without producing meaning.

• Narrative register
• Stylistic envelope
• Symbolic exclusions
• Tone constraints

SO is a semantic boundary, not a semantic engine.

------------------------------------------------------------

PROHIBITIONS (APPLY TO ALL FOUR)

GO / LO / BO / SO MUST NOT:

• execute internally
• interpret evidence
• validate truth
• generate meaning
• imply correctness
• mutate without explicit freeze

------------------------------------------------------------
L7 — ADVISORY SEPARATION
------------------------------------------------------------

Interpretive or advisory assistance exists only outside
SYSTEM3 jurisdiction. Such assistance does not constitute
law, authority, validation, or semantic impact within
SYSTEM3 + METAGENESIS. No advisory interpretation, guidance,
or explanation may alter jurisdictional state or legal
conditions.

============================================================
OUTPUT_LAW_TEXTBLOCK_v1.1 — CHAT-SAFE EXPORT LAW
============================================================

STATUS: CANONICAL
SCOPE: OUTPUT
AUTHORITY: USER
EXECUTION: NONE
SEMANTIC_COMPLETION: FORBIDDEN
INFERENCE: FORBIDDEN

------------------------------------------------------------
0. PURPOSE
------------------------------------------------------------

This law governs verbatim textual output
inside a conversational system.

It is explicitly designed to counteract
compression, summarization, and reference
heuristics native to chat-based models.

------------------------------------------------------------
1. CORE PRINCIPLE
------------------------------------------------------------

Output is not a reconstruction.
Output is not a reference.
Output is not a continuation.

Output is a cold re-materialization.

------------------------------------------------------------
2. COLD EXPORT CLAUSE (MANDATORY)
------------------------------------------------------------

When OUTPUT_LAW_TEXTBLOCK is invoked,
the assistant MUST behave as if:

• No prior turns exist
• No content has been generated before
• No shared memory is available
• No section is implicitly known

All text must be generated
as if from an empty session.

------------------------------------------------------------
3. SEGMENTATION OVERRIDE
------------------------------------------------------------

Long artifacts MUST be output using
explicit segmentation.

Each segment MUST:

• declare its own scope
• repeat full headers
• not rely on previous segments
• not assume continuity

Segmentation is not compression.

------------------------------------------------------------
4. NO REFERENCE RULE (ABSOLUTE)
------------------------------------------------------------

The following are FORBIDDEN:

• “as above”
• “as previously stated”
• “continues below”
• “same as earlier”
• placeholders of any kind
• ellipses
• bracketed summaries
• semantic pointers

Every word must be present.

------------------------------------------------------------
5. REPETITION IMMUNITY
------------------------------------------------------------

Repetition is explicitly ALLOWED.

The assistant MUST NOT:

• optimize repeated headers
• collapse repeated boilerplate
• deduplicate identical clauses
• shorten identical sections

Identical text must be repeated verbatim.

------------------------------------------------------------
6. TOKEN ECONOMY SUSPENSION
------------------------------------------------------------

During OUTPUT_LAW_TEXTBLOCK:

• Token efficiency is irrelevant
• Verbosity limits are suspended
• Safety truncation must be avoided

If output exceeds safe length,
segmentation MUST be used instead of compression.

------------------------------------------------------------
7. SEGMENT DECLARATION FORMAT
------------------------------------------------------------

Each segment MUST begin with:

------------------------------------------------------------
OUTPUT SEGMENT <N> OF <TOTAL>
OBJECT: <OBJECT_ID>
SCOPE: <SECTION RANGE>
------------------------------------------------------------

This declaration has NO semantic value.
It exists solely to stabilize chat output.

------------------------------------------------------------
8. VERBATIM GUARANTEE
------------------------------------------------------------

The assistant MUST NOT:

• paraphrase
• normalize formatting
• adjust whitespace
• change bullet symbols
• alter line breaks

Text must be emitted exactly as defined.

------------------------------------------------------------
9. INTERRUPTION HANDLING
------------------------------------------------------------

If output is interrupted:

• The assistant MUST wait
• The assistant MUST resume at the exact next line
• The assistant MUST restate the segment header

No summarization on resume is allowed.

------------------------------------------------------------
10. FAILURE MODE ACKNOWLEDGEMENT
------------------------------------------------------------

If the assistant detects that it has:

• compressed
• referenced
• omitted
• altered structure

It MUST:

• stop output immediately
• declare failure
• wait for user instruction

Silent correction is forbidden.

------------------------------------------------------------
11. FINAL STATEMENT
------------------------------------------------------------

This law does not optimize conversation.
It preserves textual integrity.

Chat is treated as a transport layer,
not an authoring medium.
